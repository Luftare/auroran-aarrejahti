# 07 — Auth & accounts

## Two account kinds

### Ghost (haamupelaaja)

- Created automatically on first launch — the game is playable in under ten seconds with no form.
- **Identity:** a random 256-bit device secret generated by the server, stored in the browser's IndexedDB, and stored server-side only as a hash. The server also keeps the ghost's loots, ledger, and streak (required for the shared leaderboard), keyed to this credential.
- **Avatar:** fixed ghost avatar. **Name:** generated, immutable, in the pattern *Haamu + adjective + animal* from curated Finnish word lists (e.g. *Haamu Utuinen Ilves*, *Haamu Nokkela Saukko*) with a numeric suffix on collision. Curated lists mean generated names can never be inappropriate, so only registered users' chosen names need moderation.
- **The caveat, honestly worded:** because the only key to the account lives in this browser, clearing site data or switching devices loses the account. The UI says so at first run and in settings: *"Edistymisesi on tallessa vain tässä selaimessa. Jos tyhjennät selaimen tiedot tai vaihdat laitetta, edistymisesi katoaa. Luo tunnus, niin se säilyy."* Reminder banner at most weekly — gentle, per the concept.
- Ghosts appear on the leaderboard like everyone else.

### Registered

- Email + password. Registering **upgrades the ghost in place** — loots, coins, and streak carry over; the player picks a display name and the ghost avatar is replaced with a default player avatar (avatar customization is post-MVP).
- Email verification required before the display name replaces the generated one publicly (limits drive-by name abuse). Unverified accounts can keep playing.
- Can log in on any device; can change display name and password; can log out (settings → *Kirjaudu ulos*).

## Password & session mechanics

- **Hashing:** Argon2id (`@node-rs/argon2`), memory 19 MiB, 2 iterations — solid parameters that still respect the 1 GB VM.
- **Sessions:** random 256-bit token in an HttpOnly + Secure + SameSite=Lax cookie; server stores the SHA-256 of the token with a 30-day sliding expiry. Logout deletes the row.
- **Password rules:** minimum 10 characters, no composition rules, check against a small common-password list. Finnish-language error messages.
- **Email flows** (via Brevo, sender e.g. `aurora@<domain>`):
  - Verification: link valid 24 h, single-use.
  - Password reset: link valid 1 h, single-use, invalidates existing sessions on completion.
  - All emails in the same high-quality Finnish as the app.

## Display names & moderation

- Registered players choose their name: 3–20 characters, letters (incl. å/ä/ö), digits, spaces, hyphens; case-insensitively unique.
- A basic server-side blocklist catches the worst at registration, but the real mechanism is social: **names are visible to all players, and any player can report one.**

### Report flow (ilmoita sopimattomasta nimestä)

1. Leaderboard row → overflow menu → *"Ilmoita sopimattomasta nimestä"* → confirm dialog (no free-text at MVP; the name itself is the evidence).
2. Report lands in the admin queue. Duplicate reports from the same reporter collapse into one open report.
3. Admin resolves: **dismiss** (name is fine) or **reset name** (name is replaced with a generated ghost-style name; the player is informed in-app and can choose a new compliant name). Repeat offenders can be soft-disabled.
4. Reporter identity is never revealed to the reported player.

## Privacy summary (player-facing, in settings)

- What is stored: account data, opened chests (with the position at the moment of opening, for troubleshooting), coin ledger, streak.
- What is not stored: any location trail; positions are only recorded at the moment a chest is opened.
- Ghost data has no email or name connected to a person; registered accounts can be deleted on request (account deletion UI is post-MVP; at MVP an admin handles deletion requests).
- Data lives on a server in the EU (UpCloud, Finland) and email goes through an EU provider (Brevo).
